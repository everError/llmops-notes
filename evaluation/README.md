# Evaluation

프롬프트·모델·파라미터를 바꿨을 때 **품질이 나빠지지 않았는지 측정**하는 영역입니다. 일반 소프트웨어의 회귀 테스트에 해당하며, LLMOps에서 가장 성숙도 차이가 크게 나는 부분입니다.

## 평가 방식

| 방식              | 설명                                              | 적합                     |
| ----------------- | ------------------------------------------------- | ------------------------ |
| 규칙 기반         | 정규식, 포함 여부, JSON 스키마 검증               | 형식이 정해진 출력       |
| 유사도 기반       | 임베딩 유사도, ROUGE/BLEU                         | 정답이 있는 요약·번역    |
| **LLM-as-Judge**  | 강한 모델이 응답을 채점                           | 개방형 응답, 품질 평가   |
| RAG 지표          | Faithfulness, Answer Relevancy, Context Precision | RAG 파이프라인           |

## 다룰 도구

- **Ragas** — RAG 특화 평가 프레임워크
- **promptfoo** — 프롬프트 A/B 비교, CI 연동에 강점
- **DeepEval** — pytest 스타일 LLM 테스트
